**📘 Git & GitHub 용어 정리**

**📌  Git이란?**

Git은 버전 관리 시스템(VCS)이다.
코드의 변경 이력을 저장하고, 이전 상태로 되돌릴 수 있다.

**📌  GitHub란?**

GitHub는 Git을 기반으로 한 원격 저장소 서비스이다.
협업, 코드 공유, 프로젝트 관리에 사용된다.

**⚙️  초기 설정 (최초 1회)**

사용중인 컴퓨터에 git 설치

[![Git](https://img.shields.io/badge/git-%23F05033.svg?style=for-the-badge&logo=git&logoColor=white)](https://git-scm.com/) 클릭!

윈도우 검색창에 명령 프롬프트 혹은 cmd 검색 후 실행 

```
git --version 
```
입력하면 현재 git의 version 나옴 -> git설치 확인하는 과정  

```
git config --global user.name "이름"
git config --global user.email "이메일"
```
git 전역 사용자를 컴퓨터에 등록하는 과정


```
git config --list
```
사용자가 등록되었는지 확인

**📂 Repository (레포지토리)**

프로젝트가 저장되는 공간
코드, 이미지, 문서 등 모든 파일 + 변경 이력이 포함됨
쉽게 말하면 프로젝트 폴더 + 히스토리 저장소

**📂 저장소(Repository) 생성**

GitHub에서 생성
GitHub 접속, 로그인 
"New repository" 클릭
레포지토리 이름 작성  
add README.md 체크하기

**🔗  로컬 프로젝트와 GitHub 연결**

로컬프로젝트: github같은 저장소와 연결이 되어있지 않고 단순 로컬pc에 저장되어있는 프로젝트

CMD 실행

```
git init
git remote add origin https://github.com/아이디/레포이름.git
```

**🌿 Branch (브랜치)**

작업을 나누는 독립적인 흐름 
기본(default) 브랜치는 main
기능 개발, 테스트 등을 따로 진행할 때 사용

**🌿  브랜치 사용**

브랜치 생성
```
git branch 브랜치명
```
브랜치 이동
```
git checkout 브랜치명
```
브랜치 생성 + 이동
```
git checkout -b 브랜치명
```

main에서 직접 작업하지 않고 
개인 브랜치에서 반드시 작업해야함

**💾 Commit (커밋)**

변경된 내용을 저장하는 단위
현재 작업 상태를 기록하는 스냅샷 역할

```
git add .
```
변경된 파일을 저장 준비 상태로 만듦

git commit -m "커밋 메시지"

**⬆️ Push (푸쉬)**

개인 브랜치의 작업 내용을 GitHub에 업로드
```
git push origin feature/기능명
```

main이 아닌 개인 브랜치에 push 해야 한다
변경사항을 나의 브랜치에 기록하고 저장

**⬇️ Pull (풀)**

작업 시작 전 반드시 최신 코드 가져오기

```
git pull origin main
```

다른 사람 작업을 반영 후 작업 시작
만약 작업영역이 겹치면 충돌발생

**📥 Clone (클론)**

원격 저장소를 내 컴퓨터로 복사
```
git clone https://github.com/아이디/레포이름.git
```

**🔀  병합 (Merge)**
```
git checkout main
git merge 브랜치명
```
일반적으로 직접 merge 안하고 
Pull Request를 통해 진행

**📬 Pull Request (PR)**

브랜치 작업 내용을 main에 합치기 전에
코드 리뷰를 요청하는 과정 (GitHub에서 진행)

👉 흐름
**브랜치 생성 → 작업 → push → PR 생성 → 리뷰 → merge**

**👉 main 브랜치에는 직접 push하지 않는다**

**⚠️  충돌 해결 (Conflict)**
같은 파일을 동시에 수정하면 충돌 발생
충돌 파일 수정 후:
```
git add .
git commit -m "충돌 해결"
```

**📜  상태 확인**
```
git status
```

**📚  로그 확인**
```
git log
```

**🚫  .gitignore**

Git에 올리고 싶지 않은 파일을 제외

예시:

node_modules/
.env
dist/

**협업 규칙 (권장)**
커밋 메시지는 명확하게 작성
브랜치는 한명당 한개만 생성
main 브랜치는 항상 안정 상태 유지

**전체 작업 흐름 (중요해요)**
```
git pull origin main
git checkout -b feature/기능명

(작업 진행)

git add .
git commit -m "작업 내용"
git push origin feature/기능명
```
이후 GitHub에서 Pull Request 생성
